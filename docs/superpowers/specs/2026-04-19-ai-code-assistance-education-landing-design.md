# AI 코드 어시스턴스 교육 프로그램 판매 랜딩페이지 설계서

- 작성일: 2026-04-19
- 목표 사용자: 비전공 입문자
- 핵심 목표: 상담 예약 전환율 최대화
- 페이지 범위: 단일 랜딩페이지(원페이지)
- 예약 정책: 상담 예약 후 결제

## 1) 정보 구조(IA) 및 컴포넌트 구성

### 1. 히어로 섹션
- 핵심 메시지: 비전공자도 AI 코드 어시스턴트를 활용해 개발 입문 가능
- 서브 메시지: 프로그램을 통해 단기간에 실무형 학습 루트 확보
- 1차 CTA: 상담 예약

### 2. 문제 공감 섹션
- 비전공 입문자가 겪는 대표 문제 3가지 제시
- 문제를 명확히 언어화해 자기 인식 유도

### 3. 해결 방식 섹션
- 교육 방식 3단계 제시(학습-실습-피드백)
- 프로그램의 차별점과 학습 부담 완화 포인트 강조

### 4. 커리큘럼 요약 섹션
- 주차별 핵심 학습 주제 요약
- 각 단계의 산출물/도달 결과를 간결하게 명시

### 5. 신뢰 요소 섹션
- 강사 소개
- 수강생 후기
- FAQ

### 6. 전환 섹션(상담 폼)
- 페이지 중단 및 하단에 CTA 재배치
- 상담 폼 필수 입력: 이름, 연락처
- 보조 문구: 상담 후 결제 진행

## 2) 데이터 흐름 및 전환 로직

1. 사용자가 CTA 클릭
2. 동일 페이지 내 상담 폼 위치로 스크롤 이동
3. 폼 입력(이름, 연락처) 후 제출
4. 제출 성공 시 사용자에게 성공 메시지 노출
5. 리드 데이터는 관리자 확인 채널로 저장(이메일/시트/API 중 구현 시점에 확정)
6. 저장 성공 후 후속 연락 안내 문구 표시

### 전환 최적화 원칙
- 중간 섹션마다 CTA를 반복 배치해 이탈 구간 최소화
- 폼 직전 신뢰 요소를 배치해 심리적 저항 완화
- 결제 CTA는 전면에 두지 않고 상담 유도에 집중

## 3) 입력 검증, 오류 처리, 접근성

### 입력 검증
- 이름 공란 여부 검증
- 연락처 형식 최소 검증

### 오류 처리
- 제출 실패 시 단일 오류 메시지 노출
- 즉시 재시도 가능 상태 제공

### 성공 처리
- 폼 초기화
- 성공 메시지 노출
- 후속 연락 SLA 안내 문구 노출

### 접근성
- 입력 라벨 명시
- 키보드 탐색/제출 지원
- 포커스 이동이 자연스럽게 동작하도록 구성

## 4) 검증 기준(테스트 시나리오)

1. 정상 입력 후 제출 성공 및 성공 메시지 표시
2. 이름 공란/연락처 형식 오류 시 검증 메시지 표시
3. 모바일에서 CTA 클릭 시 폼 위치로 자연스럽게 이동
4. 제출 실패 시 오류 메시지 및 재시도 동작 확인

## 5) 비범위(이번 1차에서 제외)

- 다중 페이지 구조
- 온라인 즉시 결제 플로우
- 고급 마케팅 자동화(세그먼트/리타게팅)
- 심층 설문 기반 리드 스코어링

## 6) 구현 결정 사항 요약

- 단일 랜딩페이지로 구현
- 핵심 KPI는 상담 예약 전환율
- 타깃은 비전공 입문자
- 폼 필수 항목은 이름+연락처로 최소화
- 결제는 상담 이후 단계에서 진행
